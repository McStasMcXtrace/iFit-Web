looktxt: Numerical data importer

version 1.4 - Feb 2014 (c) ILL

The looktxt tool, which is part of iFit, is a tool that reads any text file, with any formatting, and extracts numerical blocks into Matlab, HDF, XML, Scilab, IDL, YAML, HTML and Octave files.
INPUT: looktxt
 OUTPUT
any text file, with numbers and comments mixed A data file containing the numerical data in a scientific file format (Matlab, HDF, XML, Scilab, IDL, YAML, HTML and Octave)

1- Text/ASCII file to Upload (max 50Mb):

2- Indicate the file format to generate:

  • Matlab: generate a script which, when read, loads the data into Matlab (c).
  • HDF4: The HDF4 format is a compact, binary storage format. Such files can be edited/viewed with e.g hdfview. IDL and Matlab also have dedicated HDF 4 browsers (see hdftool). The generated file is NeXus complient.
  • HDF5: The HDF5 format is a compact, compressed binary storage format. Such files can be edited/viewed with e.g hdfview, OpenDX, Autoplot. IDL and Matlab also have dedicated HDF browsers (see hdftool). The generated file is NeXus compliant.
  • XML: The XML format, can be made NeXus compliant.
  • Scilab: generate a script which, when read, loads the data into Scilab (c).
  • IDL: generate a script which, when read, loads the data into IDL (c).
  • YAML: Data serialization format "YAML Ain't Markup Language"
  • HTML: a web page
  • Octave: generate a script which, when read, loads the data into Octave.

3- Indicate if you wish to generate a binary format when possible (faster to read):

You will get a ZIP archive containing the initial information, the conversion log, and the generated file.

Limitations

This translator works nicely with text files, but will fail with binary files.

How it works

The initial text file is read as a human being would do, identifying numerical blocks, labeling them with the neighboring comments. The output file is then generated into a scientific data format.
