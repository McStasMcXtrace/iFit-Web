Freshly Cooked Virtual Machines

version 1.0 - Aug 2019 (c) SOLEIL

This page allows to select and launch a Virtual machine in a Browser Window, with distant execution. This way, you can access pre-installed systems with specific applications matching scientific domains.

INPUT: VirtualBox machine
 OUTPUT
Select the machine you need
 A browser showing the distant machine display.


1- Indicate the machine you need:
  • Damn Small Linux: An incredibly small linux distribution from DSL (4.4).

2- Indicate if you need a persistent session

  • A non persistent session is cleaned when you either shutdown the virtual machine, or close the browser tab.
  • A persistent session keeps running until you fully shutdown the virtual machine. You may close the browser, and login again later using the same ID token sent by email. Persistent sessions may be required for e.g. long calculations.


3- Indicate your email address:
Required, to get the connection information. Without it, you will NOT be able to login. Use name@domain.name

A virtual machine will be created, with assigned resources (e.g. 10Gb disk, 4 CPU's, 4 Gb memory). You will receive an email with the connection directives. A browser window will be opened. Use the given ID token to connect to the VNC.You may change the screen resolution of the virtual machine using the bottom-left menu Preferences/Monitor Settings.

Limitations

The virtual machines are made secure by inactivating some of their functionalities. Remember that the virtual machine is created on request, and destroyed afterwards. You should then export elsewhere any work done there-in (e.g. mounted disk, ssh/sftp, DropBox, ownCloud, ...). This is a single-shot virtual machine.

We recommend to use the Firefox web browser to display the virtual machine. Google Chrome is also OK. Other web browsers may show an offset between two mouse pointers (local and distant).

This is an experimental service, and as such, it may be restarted for updates. We shall inform, as much as possible, logged users about any service interruption.

How it works

A one-shot virtual machine is cloned from a template. It is then started with QEMU/KVM. A message is sent to your email address with a link and a connection token, which is one-shot for non-persistent sessions. A VNC client (noVNC) is connected to it, and shown in your browser. All the available virtual machines are stored on our Apache server (ISO and QCOW2 files).
