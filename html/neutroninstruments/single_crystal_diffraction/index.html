sqw_phonons: Phonon dispersions in 4D

version 1.8-1 - E. Farhi, Apr 2016

The sqw_phonons tool, which is part of iFit, is a tool to compute the phonon dispersion curves (vibrational modes) in 4D (HKLE) and for any crystalline material. It only requires as input a file describing the material structure, and one can select a few key parameters for the ab-initio DFT codes. The details of the sqw_phonons capabilities are explained in the iFit Models page.

To try this service, select use Aluminium, then use the EMT code, which is extremely fast (but very limited), and press the COMPUTE button. You should get something like this.
INPUT: sqw_phonons
 OUTPUT
A POSCAR, CIF, PDB, ... material structure Phonon dispersions from the Dynamical matrix, using forces estimated by ab-initio using ASE and a selection of DFT codes (EMT, GPAW, ABINIT, Elk, Dacapo, QuantumEspresso).

1- Material structure (max 5Mb):
Any file format supported by the ASE can be used, such as (POSCAR, CIF, PDB, ETSF, MDL/SDF/XSD, ...).
See the full list here. We provide a few examples: Al, AlN, MgO, Si, NaCl, SrTiO3.

2- Indicate the calculator to use:
We recommend QuantumEspresso (or EMT with a simple metal for a try).

3- Indicate if the material is a conductor or insulator:
If you don't know, leave the smearing on, to help convergence.

4- Indicate the energy cut-off:
Nice people say the cut-off energy should not be lower than 1000 eV.
However, this results in very long computations. To start you can use 260 or 340 eV.

5- Indicate the K-points binning:
Using a Monkhorst-Pack K-points mesh 3x3x3 is the minimum for sensible results. You can try smaller as a test, but this will probably give you strange results.

6- Indicate the supercell size:
Using a supercell 3x3x3 is often the minimum for sensible results. For large material structures (containing many atoms), you may try 2x2x2.

7- Indicate your email address:
Recommended. If you are behind a proxy server, do NOT use this option.

What will happen

The input material description will be processed and sent to the selected DFT code. The structure will first be optimized. Then, the atoms will be moved slightly, and the restoring forces will be computed. The dynamical matrix can then be evaluated for any HKL location in the crystal reciprocal space. We then compute the phonon spectrum (vibrational density of states), and the dispersions in a full HKL cube, as example. You will be able to evaluate the Model anywhere yourself. by evaluating the resulting Model using iFit.

In addition:
  • You will be notified by email when the computation starts and ends.
  • A full report will be sent to you upon completion as a ZIP file. Open the 'index.html' document there-in.
  • An example of such a report using QuantumEspresso with Aluminium is available here.
  • The archive contains input data files, log files, images (PNG, VTK, MRC, FIG/MAT, ...).
  • The Model is provided as a Matlab object, to be used within iFit, which evaluation is very fast, without the need to restart the DFT.
  • You can read and analyze the results with iFit.

Limitations

There is no guaranty that the phonon dispersions that will be computed are exact. The validity depends on the material, the DFT code, and the peudo-potentials in use. In practice, if you get zero phonon energy outside Bragg locations, this may be an indication that the Model is not satisfactory. You should then increase the supercell size, the k-points mesh, and last the cut-off energy.

Currently, this service only computes the vibrational modes energy, not their intensity and polarisation vectors.
